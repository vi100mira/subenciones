# Product Positioning: Public Radar and Private Opportunity RAG

## Decision

The product value starts with an AI-iterative public opportunity radar. Entities should get useful searches, explanations, alerts, and prioritization using only a minimal profile and the platform-managed public corpus.

## Private RAG Scope

Tenant-private RAG is optional and must focus on approved opportunity sources:

- calls received by email
- PDFs or bases from funders
- Drive/SharePoint folders limited to opportunities
- federation or partner alerts
- historical opportunities
- approved facts that improve matching

It is not a default ingestion of all company/entity data.

## Product Implication

The onboarding promise is low-friction and low-risk: start with minimal data, search the platform radar, then add private opportunity sources only if the entity wants more precision.
